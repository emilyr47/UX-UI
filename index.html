About

Welcome to my Final Digital Project for CI 4312. Throughout this semester, I have taken a special interest in usability, and, specifically how human factors impact and interact with the web. This project is centered around the ten heuristics of user interface (UI) design, written and developed by Jakob Nielsen. Each of these ten heuristics plays an important role in ensuring accessibility and usability on the web. In this project, I will demonstrate each of these ten heuristics by explaining them and then providing applicable examples of how they can be exercised in UI design and can imporve user experience (UX).



Visibility

Visibility informs users of system status by providing feedback. This could look like a spinning loader as a program loads, a progress bar gradually filling during an update, or any visual or form of feedback that tells the user that the request is working. A simple example of this is shown in the buttons below.

Button #1: No Visibility or Feedback

  • No change in cursor on hover
  • No color change in button on hover
  • No text indicating where the button is going or if it is going to where the user wants
  • No link appears on hover


Button #2: Good Visibility & Feedback

  • Cursor changes to a pointer/hand on hover
  • Button slightly changes color on hover
  • Button text tells the user where the button leads
  • Button's link appears in bottom left corner (location may differ per web browser)
